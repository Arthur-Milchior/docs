  <div id="readme" class="Box-body readme blob js-code-block-container p-5 p-xl-6 gist-border-0">
    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-release--version-identifier" class="anchor" aria-hidden="true" href="#release--version-identifier"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Release &amp; Version Identifier</h1>
<p>Jina is shipped from two package management systems, PyPi and Docker Hub. This article clarifies the release cycle and version identifier behind each system.</p>


<ul>
<li><a href="#pypi-package-versioning">PyPi package versioning</a></li>
<li><a href="#docker-image-versioning">Docker image versioning</a></li>
<li><a href="#manual-release-entrypoint">Manual Release Entrypoint</a></li>
</ul>

<h2><a id="user-content-pypi-package-versioning" class="anchor" aria-hidden="true" href="#pypi-package-versioning"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>PyPi package versioning</h2>
<p>We follow the <a href="https://semver.org/" rel="nofollow">semantic versioning</a> and <a href="https://www.python.org/dev/peps/pep-0440/" rel="nofollow">PEP-440</a>. Jina's version is identified by <code>x.y.z</code> (i.e. "major.minor.patch").</p>
<p>To install the latest final release:</p>
<div class="highlight highlight-source-shell"><pre>pip install -U jina</pre></div>
<p>To install a particular final release:</p>
<div class="highlight highlight-source-shell"><pre>pip install jina==x.y.z</pre></div>
<p>The term "final release" is relative to "developmental release" as described below.</p>
<h3><a id="user-content-developmental-releases-versioning" class="anchor" aria-hidden="true" href="#developmental-releases-versioning"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Developmental releases versioning</h3>
<p>One every master-merging event, we create early releases directly from source control which do not conflict with later project releases. The version identifier for development release is <code>x.y.z.devN</code>, which adds a suffix <code>.dev</code>, followed by a non-negative integer value <code>N</code>, which is reset on every release.</p>
<p>To install the latest development release:</p>
<div class="highlight highlight-source-shell"><pre>pip install --pre jina</pre></div>
<h3><a id="user-content-version-epochs" class="anchor" aria-hidden="true" href="#version-epochs"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Version epochs</h3>

<h2><a id="user-content-docker-image-versioning" class="anchor" aria-hidden="true" href="#docker-image-versioning"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Docker image versioning</h2>
<p>Our univeral Docker image is ready-to-use on linux/amd64, linux/armv7+, linux/arm/v6. The Docker image name always starts with <code>jinaai/jina</code> followed by a tag composed of three parts:</p>
<pre lang="text"><code>jinaai/jina:{version}{python_version}{extra}
</code></pre>
<ul>
<li><code>{version}</code>: The version of Jina. Possible values:
<ul>
<li><code>latest</code>: the last release;</li>
<li><code>master</code>: the master branch of <code>jina-ai/jina</code> repository;</li>
<li><code>x.y.z</code>: the release of a particular version;</li>
<li><code>x.y</code>: the alias to the last <code>x.y.z</code> patch release, i.e. <code>x.y</code> = <code>x.y.max(z)</code>;</li>
</ul>
</li>
<li><code>{python_version}</code>: The Python version of the image. Possible values:
<ul>
<li><code> </code>, <code>-py37</code>: Python 3.7;</li>
<li><code>-py38</code> for Python 3.8;</li>
<li><code>-py39</code> for Python 3.9;</li>
</ul>
</li>
<li><code>{extra}</code>: the extra dependency installed along with Jina. Possible values:
<ul>
<li><code> </code>: Jina is installed inside the image via <code>pip install jina</code>;</li>
<li><code>-devel</code>: Jina is installed inside the image via <code>pip install jina[devel]</code>;</li>
<li><code>-daemon</code>: Jina is installed inside the image via <code>pip install jina[dameon]</code> along with <code>fluentd</code>; <strong>and the entrypoint is set to <code>jinad</code></strong>.</li>
</ul>
</li>
</ul>
<p>Examples:</p>
<ul>
<li><code>jinaai/jina:0.9.6</code>: the <code>0.9.6</code> release with Python 3.7 and the entrypoint of <code>jina</code>.</li>
<li><code>jinaai/jina:latest-py38-daemon</code>: the latest release with Python 3.8 base and the entrypoint of Jina daemon.</li>
<li><code>jinaai/jina:latest</code>: the latest release with Python 3.7 and the entrypoint of <code>jina</code></li>
<li><code>jinaai/jina:master</code>: the master with Python 3.7 and the entrypoint of <code>jina</code></li>
</ul>
<h3><a id="user-content-do-i-need--devel" class="anchor" aria-hidden="true" href="#do-i-need--devel"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Do I need <code>-devel</code>?</h3>
<p>Use <code>-devel</code> image, if you want to:</p>
<ul>
<li>have efficiency improvement on AsyncIO and data compression</li>
<li>enable prettified error printing</li>
<li>build Jina Hub extension</li>
<li>expose REST interface beyond gRPC</li>
<li>enable log-streaming/aggregating via <code>fluentd</code></li>
<li>enable mime-type sniffing</li>
</ul>
<h3><a id="user-content-image-alias-and-updates" class="anchor" aria-hidden="true" href="#image-alias-and-updates"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Image alias and updates</h3>

<h3><a id="user-content-image-size-on-different-tags" class="anchor" aria-hidden="true" href="#image-size-on-different-tags"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Image size on different tags</h3>
